## Changelog · NeoIni

### 1.8-pre1 — 15 марта 2026

#### List of changes

- Исправлено отображение экранирования в методе `Search`.
- Устранены лишние аллокации памяти.
- Добавлена опциональная поддержка значений в кавычках через параметр `UseShielding` (например, `key = "value ; not a comment"`).
- Проведен рефакторинг кода и удалено дублирование.

### 1.7.3 — 15 марта 2026

#### List of changes

- Добавлен интерфейс INeoIniProvider для подключаемых хранилищ (база данных, удалённый сервис, память и т.д.)
- Реализован NeoIniFileProvider, инкапсулирующий всю существующую файловую логику
- Добавлены конструкторы NeoIniReader, принимающие INeoIniProvider (sync, async, human mode)
- Добавлено исключение UnsupportedProviderOperationException для файловых операций на кастомных провайдерах
- Защита UseAutoBackup в ApplyOptions для предотвращения крэша на нефайловых провайдерах
- NeoIniData сделан публичным для возможности реализации кастомных провайдеров

### 1.7.2 — 13 марта 2026

#### List of changes

- Исправлена уязвимость при передаче значений в `Set`-методы

### 1.7.2-pre1 — 13 марта 2026

#### List of changes

- Добавлен режим ручного редактирования INI (HumanMode) с автоматическим сохранением и восстановлением комментариев
- Добавлен опциональный конвейер «humanization», сохраняющий позиции комментариев относительно секций и ключей
- Валидация контрольной суммы автоматически отключается при включённом HumanMode для возможности ручных правок
- Запрещено одновременное использование HumanMode и шифрования для предотвращения повреждения данных

### 1.7.1 — 13 марта 2026

#### List of changes

- Добавлена функция горячей перезагрузки через наблюдатель за файлом.

### 1.7 — 12 марта 2026

#### List of changes

- Удалён оставшийся и неиспользуемый код из NeoIniMappingGenerator для упрощения реализации source-генератора.

### 1.7-pre1 — 12 марта 2026

#### List of changes

- Добавлен мини-пакет NeoIni.Annotations с NeoIniKeyAttribute для маппинга свойств модели на конкретные INI-секции и ключи, включая опциональный DefaultValue.
- Добавлен мини-пакет NeoIni.Generators с NeoIniMappingGenerator (IIncrementalGenerator), который сканирует свойства с атрибутом NeoIniKeyAttribute и генерирует NeoIniReaderExtensions со строго типизированными Get / Set API для маппинга объектов.
- Генерируемый NeoIniReaderExtensions.Get создаёт и заполняет экземпляры конфигурации из INI-файлов через GetValue для каждого маппированного свойства, применяя значения по умолчанию при отсутствии значений.
- Генерируемый NeoIniReaderExtensions.Set записывает экземпляры конфигурации обратно в INI-файлы через SetValue, выбрасывая NotSupportedException для типов, не покрытых маппингом source-генератора.

### 1.6.1 — 12 марта 2026

#### List of changes

- Исправлено экранирование обратного слэша (`\\`) в парсере для предотвращения повреждения путей Windows
- Исправлена проблема, при которой сохранение полностью пустой конфигурации (все секции удалены) игнорировалось
- Исправлена логика парсинга в `TryMatchKey` для корректной загрузки ключей с пустыми значениями при включённом AllowEmptyValues
- Добавлена null-безопасность (`?? string.Empty`) в `SaveFile` и `SaveFileAsync` для предотвращения `ArgumentNullException`
- Улучшено управление памятью `CryptoStream` через `leaveOpen: true` для предотвращения преждевременного закрытия потока

### 1.6.1-pre2 — 12 марта 2026

#### List of changes

- Исправлен расчёт минимальной длины файла в `NeoIniFileProvider` с учётом `SaltSize`
- Реализован автоматический фолбэк получения пароля при открытии автозашифрованного файла с пользовательским паролем

### 1.6.1-pre1 — 12 марта 2026

#### List of changes

- Рефакторинг делегатов `Action` на `EventHandler`
- Добавлено поле `AllowEmptyValues` для управления пустыми значениями данных
- Улучшена общая оптимизация кода и производительность выполнения
- Добавлены новые clamped API-методы:
  - `AddKeyClamped<T>` (`AddKeyClampedAsync<T>`)
  - `SetValueClamped<T>` (`SetValueClampedAsync<T>`)
- Переименованы API-методы для единообразия: `AddKeyInSection` -> `AddKey` (включая async-варианты) и `SetKey` -> `SetValue` (включая async-варианты)

### 1.6.0.1 — 6 марта 2026

#### List of changes

- Исправлено присвоение флага hasSalt

### 1.6 — 6 марта 2026

#### List of changes

- Переработан конвейер шифрования: PBKDF2 с уникальным per-file salt вместо salt, производного от пароля
- Случайный 16-байтный salt встроен в формат файла рядом с IV для зашифрованных конфигураций
- Обновлено получение ключа для авто-шифрования (на основе окружения) и пользовательского пароля
- Добавлен вспомогательный класс NeoIniIO для централизации буферизованных операций чтения/записи
- Переписан async file IO с использованием настоящих асинхронных операций FileStream с поддержкой отмены
- Добавлен путь асинхронной загрузки данных в NeoIniFileProvider и NeoIniReader
- Реализованы фабричные перегрузки CreateAsync (plain, auto-encrypted, password-encrypted) для неблокирующей инициализации
- Исправлены и унифицированы async-варианты основных публичных API-методов (GetValueAsync, SetKeyAsync, операции с секциями/ключами)
- Улучшена потокобезопасность и обработка отмены в async-методах (использование lock + проверки CancellationToken)

---

> **1.5.x и ниже** — ранняя активная разработка. Библиотека формировалась: API часто менялся, баги были обычным делом, многие релизы — инкрементальные фиксы, а не запланированные фичи. Стабильное использование начинается с 1.6.
