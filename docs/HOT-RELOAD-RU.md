## Hot Reload Support (1.7.1+)

Начиная с **1.7.1**, NeoIni добавляет механизм **hot‑reload** для автоматической перезагрузки конфигурационных файлов при изменении на диске — без перезапуска приложения.

Это полезно для долгоживущих сервисов, дебаг‑сессий и дев‑инструментов, которым нужно динамически реагировать на живые изменения конфигурации.

### Starting and stopping hot reload

Вы можете запустить мониторинг INI‑файла в реальном времени, вызвав `StartHotReload(int delayMs)`.

Этот метод с указанным интервалом (в миллисекундах) проверяет файл на изменения и перезагружает его при обнаружении модификации.

```csharp
NeoIniReader reader = new("config.ini");

// Запускаем hot reload с опросом раз в 2 секунды
reader.StartHotReload(2000);

// Останавливаем отслеживание изменений
reader.StopHotReload();
```

### Behavior and safety

- Минимальная задержка опроса — **1000 ms**. В противном случае будет выброшено `InvalidHotReloadDelayException`.  
- Процесс hot reload работает асинхронно в фоне, проверяя контрольные суммы файла.  
- На один экземпляр NeoIniReader может быть запущен только один watcher.  
- При каждом обнаруженном изменении NeoIni автоматически вызывает `ReloadFromFileAsync()` для обновления данных конфигурации в памяти.

> **Note:** Оверхед от этой функции минимален, так как используются сравнение checksum и cancellation tokens для аккуратной остановки.
