## Hot-reload (2.0+)

Автоматическое обнаружение изменений конфигурации и перезагрузка данных без перезапуска приложения. NeoIni использует polling с проверкой контрольной суммы для отслеживания внешних модификаций INI-источника.

---

### Запуск и остановка (встроенный монитор)

```csharp
using NeoIni;

NeoIniDocument document = new("config.ini");

// Запуск hot-reload с polling каждые 2 секунды (встроенный файловый монитор)
document.StartHotReload(2000);

// Остановка отслеживания
document.StopHotReload();
```

`StartHotReload(int pollingInterval)` принимает интервал опроса в миллисекундах. Для встроенного файлового монитора минимум составляет **1000 мс** – меньшие значения вызывают `InvalidHotReloadDelayException`. Вызовите `StopHotReload()` для отключения в любой момент.

---

### Пользовательские мониторы (2.0+)

Версия 2.0 вводит интерфейс `IHotReloadMonitor`, позволяющий реализовать собственную логику обнаружения изменений (например, с помощью `FileSystemWatcher`, уведомлений от базы данных или очередей сообщений).

```csharp
public interface IHotReloadMonitor : IDisposable
{
    event EventHandler? ChangeDetected;
    void Start(int pollingInterval);
    void Pause();
    void Continue();
    Task ContinueAsync();
    void Stop();
}
```

Передайте свой монитор вторым параметром в `StartHotReload`:

```csharp
document.StartHotReload(2000, new MyCustomMonitor());
```

По умолчанию используется встроенный файловый монитор (если параметр опущен). Пользовательские мониторы могут определять собственную семантику для `pollingInterval` – ограничение в 1000 мс действует **только для стандартного файлового монитора**.

---

### Поведение и безопасность

- **Polling + контрольная сумма:** На каждом интервале монитор вычисляет контрольную сумму текущего состояния источника и сравнивает её с последним известным значением. При различии возникает событие `ChangeDetected`, и документ автоматически перезагружается.
- **Потокобезопасность:** Перезагрузка захватывает внутренний write lock `AsyncReaderWriterLock`, поэтому конкурентное чтение безопасно.
- **События:** Успешная перезагрузка вызывает событие `Loaded`. При ошибке срабатывает событие `Error`, а предыдущие данные сохраняются.
- **Приостановка при сохранении:** Монитор автоматически приостанавливается во время сохранения документа, чтобы не инициировать перезагрузку при собственной записи.

---

### Поддержка провайдеров

Hot-reload работает с любым `INeoIniProvider`, который возвращает осмысленное значение из `GetStateChecksum()`. Встроенный `NeoIniFileProvider` использует временну́ю метку последней записи и размер файла в качестве контрольной суммы. Пользовательские провайдеры могут возвращать любой массив байт, изменяющийся при изменении данных.

> **Примечание:** В версиях до 2.0 hot‑reload был встроен непосредственно в `NeoIniReader` и не поддерживал пользовательские мониторы. Перейдите на версию 2.0, чтобы воспользоваться новой расширяемой архитектурой.
